#  Tsumige-Kaishou Bot

ゲーマー共通の課題である**「積みゲー（購入したが未プレイのゲーム）」**を解消するため、Discordコミュニティ内でのゲーム所持・プレイ状況を可視化・比較するDiscordボットです。

Steam Web APIから取得したリアルなプレイデータを基に、「自分だけが持っている隠れた名作」や「サーバー内にいる既プレイの先輩」を自動で抽出し、チャット上での新たなコミュニケーションを創出します。

---

##  アーキテクチャと技術スタック

将来的な機能拡張とメンテナンス性を重視し、**ヘキサゴナルアーキテクチャ（Ports and Adapters）**を採用しています。インフラ層（DBや外部API）を疎結合に保つことで、コアなロジックを保護し、変更に強い設計を実現しました。


* **Language:** TypeScript
* **Library:** Discord.js v14
* **ORM:** Prisma v7
* **Database:** PostgreSQL
* **External API:** Steam Web API

---

## 🚀 現在の機能 (MVP)

現在は最小限の機能（Minimum Viable Product）として、以下のフローを実装しています。

-   **`/register`**: SteamID64（17桁の識別子）をDiscordアカウントと紐付けます。
-   **`/sync`**: Steam API経由で全所有ゲームのタイトル・画像・プレイ時間をDBへ保存・更新します。
-   **`/tumige`**: サーバー内の他ユーザーとプレイ状況を比較し、結果をEmbed形式で表示します。
    -   **布教モード**: 自分だけが持っているゲームを抽出。
    -   **先輩発見モード**: そのゲームをクリア済みのユーザーを表示し、アドバイスを仰げる状態に。
    -   ※Discord APIの制限に基づき、現在は上位10件を自動抽出して表示します。

## 🛠️ 今後の改良予定

### 1. データベース同期処理の並列化
現在は大量のゲームデータを1件ずつ直列に保存（`await`のループ）していますが、これを `Promise.all` を用いた並列実行へ移行予定です。数百本のゲームを持つユーザーの同期時間を大幅に短縮し、UXの向上を図ります。

### 2. データクレンジング機能の実装
現在は「新規追加・更新」のみを行っていますが、Steam側で削除または非公開に設定されたゲーム

### 3. Steam APIを用いたリアルタイム検証の強化
`/register` 実行時に、実際にSteam APIへリクエストを送り、「プロフィールの公開設定が正しいか」「実在するIDか」を即時に検証する仕組みを導入します。

### 4. インタラクティブなUIへの拡張
Discordのボタン機能（Components）を活用し、10件以上の積みゲーをページ送りで確認できる**ページネーション機能**を実装予定です。
